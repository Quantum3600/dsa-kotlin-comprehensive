## üöÄ Quick Start for Contributors

Want to help? Here's how to start:

### 1. Pick a Category
Choose from unimplemented categories based on your interest and the current phase.

### 2. Follow the Template
Use any of the 10 implemented files as a template:
- LargestElement.kt (arrays)
- BubbleSort.kt (sorting)
- BinarySearch.kt (searching)
- FibonacciNumber.kt (recursion)
- ClimbingStairs.kt (DP)
- etc.

### 3. Implementation Checklist
- [ ] Problem statement (300-500 words)
- [ ] Approach & intuition (500-800 words)
- [ ] Visual examples with ASCII
- [ ] Multiple approaches
- [ ] Complexity analysis (300-400 words)
- [ ] Line-by-line comments
- [ ] Example walkthrough (400-600 words)
- [ ] 8-10 edge cases
- [ ] When to use / real-world applications
- [ ] Common mistakes
- [ ] 8-10 test cases in main()

### 4. Submit PR
Follow guidelines in [CONTRIBUTING.md](CONTRIBUTING.md)
